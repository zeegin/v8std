# Временная таблица должна содержать индексы (ql-temp-table-index)

- Категория: `ql`

1. Индекс следует строить если: 	1.1 Большая временная таблица участвует в соединении (не важно, с какой стороны).  	    В индекс следует добавлять поля, участвующие в условии ПО. 	1.2 Обращение к временной таблице выполняется в подзапросе конструкции логического оператора В (...).  	    В индекс следует добавлять поля временной таблицы из списка выбора, соответствующие перечисленным  	    с левой стороны логического оператора В (...). 2. Маленькие временные таблицы индексировать не нужно (менее 1000 записей).
