# Автоформатирование кода

Внешняя обработка «Автоформатирование кода и локализация» изменяет
XML-исходники конфигурации по выбранным правилам. Она может сама выгрузить
конфигурацию из информационной базы, показать сравнение, загрузить выбранные
изменения обратно и работать с хранилищем конфигурации.

!!! warning "Проверяйте изменения перед загрузкой"

    Обработка исправляет исходники, а не только сообщает об отклонениях.
    Запускайте её на рабочей копии и просматривайте сравнение файлов.

## Где получить

- [Скачать актуальную обработку с ИТС](https://its.1c.ru/db/files/1CITS/EXE/V8Std/АвтоформатированиеКодаИЛокализация/АвтоформатированиеКодаИЛокализация.zip).

## Пакетный запуск

Настройте обработку интерактивно и выполните команду
**Выгрузить параметры автоформатирования**. Она создаёт XML или ZIP с точной
структурой параметров текущей версии.

=== ":fontawesome-brands-windows: Windows"

    ```bat
    "C:\Program Files\1cv8\8.3.27.XXXX\bin\1cv8.exe" ENTERPRISE ^
      /F "C:\Bases\Development" ^
      /N "Администратор" ^
      /Execute "C:\Tools\АвтоформатированиеКодаИЛокализация.epf" ^
      /C "C:\Tools\Config.xml"
    ```

=== ":fontawesome-brands-apple: macOS / :fontawesome-brands-linux: Linux"

    ```bash
    /opt/1cv8/x86_64/8.3.27.XXXX/1cv8 ENTERPRISE \
      /F "/Users/user/Bases/Development" \
      /N "Администратор" \
      /Execute "/Users/user/Tools/АвтоформатированиеКодаИЛокализация.epf" \
      /C "/Users/user/Tools/Config.xml"
    ```

### Примеры `Config.xml`

=== "Файловая база"

    Конфигурация выгружается во временный каталог, форматируется и загружается
    обратно в файловую базу. После загрузки обновляется конфигурация базы данных.

    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <НастройкиАвтоформатирования>
      <ТипИсточника>0</ТипИсточника>
      <ПутьКФайловойБазе>C:\Bases\Development</ПутьКФайловойБазе>
      <Пользователь>Администратор</Пользователь>
      <Пароль></Пароль>
      <КаталогВыгрузкиXML>C:\Temp\autoformat</КаталогВыгрузкиXML>
      <СтрокаЗапускаПлатформы>C:\Program Files\1cv8\8.3.27.XXXX\bin\1cv8.exe</СтрокаЗапускаПлатформы>
      <ОбновитьКонфигурациюБД>true</ОбновитьКонфигурациюБД>
    </НастройкиАвтоформатирования>
    ```

=== "Серверная база"

    Обработку нужно запускать в этой же серверной базе: значение
    `ПутьКБазе` в XML не переключает базу, в которой уже запущен пакетный
    сеанс.

    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <НастройкиАвтоформатирования>
      <ТипИсточника>1</ТипИсточника>
      <ПутьКБазе>Srvr="server";Ref="development";</ПутьКБазе>
      <Пользователь>Администратор</Пользователь>
      <Пароль></Пароль>
      <КаталогВыгрузкиXML>C:\Temp\autoformat</КаталогВыгрузкиXML>
      <СтрокаЗапускаПлатформы>C:\Program Files\1cv8\8.3.27.XXXX\bin\1cv8.exe</СтрокаЗапускаПлатформы>
      <ОбновитьКонфигурациюБД>true</ОбновитьКонфигурациюБД>
    </НастройкиАвтоформатирования>
    ```

=== "Проект в хранилище"

    Перед автоформатированием обработка обновит конфигурацию из хранилища.
    При значении `true` параметра
    `ПомещатьИзмененныеОбъектыВХранилище` она захватит объекты, поместит
    изменения с указанным комментарием и отменит оставшиеся захваты. Значение
    `false` оставит изменения только в базе.

    В исследованной версии все правила выбраны по умолчанию. Если не добавлять
    секцию `СтандартыДляАвтоформатирования`, обработка оставит включённым весь
    набор правил:

    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <НастройкиАвтоформатирования>
      <ТипИсточника>0</ТипИсточника>
      <ПутьКФайловойБазе>C:\Bases\Development</ПутьКФайловойБазе>
      <Пользователь>Администратор</Пользователь>
      <Пароль></Пароль>
      <КаталогВыгрузкиXML>C:\Temp\autoformat</КаталогВыгрузкиXML>
      <СтрокаЗапускаПлатформы>C:\Program Files\1cv8\8.3.27.XXXX\bin\1cv8.exe</СтрокаЗапускаПлатформы>
      <КаталогХранилища>tcp://repository-server/repository</КаталогХранилища>
      <ПользовательХранилища>developer</ПользовательХранилища>
      <ПарольХранилища></ПарольХранилища>
      <ПомещатьИзмененныеОбъектыВХранилище>true</ПомещатьИзмененныеОбъектыВХранилище>
      <ТекстКомментария>Автоформатирование кода</ТекстКомментария>
      <ОбновитьКонфигурациюБД>true</ОбновитьКонфигурациюБД>
    </НастройкиАвтоформатирования>
    ```

=== "XML-выгрузка"

    Обработка изменит существующую XML-выгрузку непосредственно в указанном
    каталоге. В нём должен находиться файл `Configuration.xml`.

    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <НастройкиАвтоформатирования>
      <ТипИсточника>2</ТипИсточника>
      <КаталогВыгрузкиXML>C:\Projects\Configuration</КаталогВыгрузкиXML>
      <КаталогВыгрузкиРезультат>C:\Projects\Configuration</КаталогВыгрузкиРезультат>
      <СтрокаЗапускаПлатформы>C:\Program Files\1cv8\8.3.27.XXXX\bin\1cv8.exe</СтрокаЗапускаПлатформы>
    </НастройкиАвтоформатирования>
    ```

=== "Проект EDT"

    Укажите корень проекта EDT, содержащий `src/Configuration/Configuration.mdo`.
    Обработка изменит файлы проекта непосредственно.

    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <НастройкиАвтоформатирования>
      <ТипИсточника>2</ТипИсточника>
      <КаталогВыгрузкиXML>C:\Projects\MyEDTProject</КаталогВыгрузкиXML>
      <КаталогВыгрузкиРезультат>C:\Projects\MyEDTProject</КаталогВыгрузкиРезультат>
      <СтрокаЗапускаПлатформы>C:\Program Files\1cv8\8.3.27.XXXX\bin\1cv8.exe</СтрокаЗапускаПлатформы>
    </НастройкиАвтоформатирования>
    ```

=== "Настройка отбора"

    Если секция `СтандартыДляАвтоформатирования` присутствует, обработка
    сначала отключает все правила, затем включает перечисленные. Родительский
    ключ `СтандартNNN` обязателен: без него обработчик стандарта не запустится.

    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <НастройкиАвтоформатирования>
      <ТипИсточника>2</ТипИсточника>
      <КаталогВыгрузкиXML>C:\Projects\Configuration</КаталогВыгрузкиXML>
      <КаталогВыгрузкиРезультат>C:\Projects\Configuration</КаталогВыгрузкиРезультат>
      <СтрокаЗапускаПлатформы>C:\Program Files\1cv8\8.3.27.XXXX\bin\1cv8.exe</СтрокаЗапускаПлатформы>
      <СтандартыДляАвтоформатирования>
        <!-- Весь стандарт 455: родитель и все его правила. -->
        <Стандарт>Стандарт455</Стандарт>
        <Стандарт>ЗаменаОбластейВВидеКомментариев</Стандарт>
        <Стандарт>УдалениеПустыхОбластей</Стандарт>
        <Стандарт>ДобавлениеОбластиОписаниеПеременных</Стандарт>
        <Стандарт>ДобавлениеОбластиИнициализация</Стандарт>

        <!-- Только п. 1.3 стандарта 455: удалите правила выше и оставьте:
        <Стандарт>Стандарт455</Стандарт>
        <Стандарт>ЗаменаОбластейВВидеКомментариев</Стандарт>
        -->
      </СтандартыДляАвтоформатирования>
    </НастройкиАвтоформатирования>
    ```
