# Автоформатирование кода и локализация

Внешняя обработка «Автоформатирование кода и локализация» изменяет
XML-исходники конфигурации по выбранным правилам. Она может сама выгрузить
конфигурацию из информационной базы, показать сравнение, загрузить выбранные
изменения обратно и работать с хранилищем конфигурации.

!!! warning "Проверяйте изменения перед загрузкой"

    Обработка исправляет исходники, а не только сообщает об отклонениях.
    Запускайте её на рабочей копии и просматривайте сравнение файлов.

## Интерактивный запуск

1. В режиме 1С:Предприятие выберите **Файл → Открыть** и откройте EPF.
2. Выберите источник, правила и состав объектов.
3. Выполните автоформатирование.
4. Сравните исходные и изменённые файлы.
5. Загрузите только проверенные изменения.

Можно включить все правила, отключить все либо выбрать отдельные
преобразования.

## XML-выгрузка и проект EDT

Для готовых исходников выберите источник **Выгрузка XML** и укажите корневой
каталог. Обработка работает с XML-выгрузкой Конфигуратора и распознаёт проект
EDT по каталогу `src`. Состав можно ограничить списком файлов, включёнными
подсистемами и исключёнными подсистемами.

Если автоматическая выгрузка отключена, обработка изменяет файлы указанного
каталога. После выполнения сравните результат средствами контроля версий.

## Файловая и серверная база

Для файловой базы укажите каталог с `1Cv8.1CD`. Для серверной — кластер
серверов 1С:Предприятия и имя информационной базы; обработка формирует строку:

```text
Srvr="<сервер>";Ref="<информационная база>"
```

Укажите путь к `1cv8.exe`, пользователя, пароль и каталог временной
XML-выгрузки. Отдельными флагами разрешаются:

- выгрузка конфигурации в XML;
- загрузка изменённого XML обратно;
- обновление конфигурации базы данных.

## Работа с хранилищем

Укажите каталог, пользователя и пароль хранилища. Без флага помещения
изменений обработка только получает конфигурацию из хранилища.

С включённым флагом она:

1. подключает конфигурацию к хранилищу;
2. пытается захватить выбранные объекты;
3. выгружает их в XML и выполняет преобразования;
4. загружает проверенные XML-файлы;
5. помещает изменённые объекты в хранилище;
6. отменяет оставшиеся захваты.

Объекты, которые не удалось захватить, отмечаются отдельно.

## Результат

Форма результата позволяет сравнить файл до и после преобразования, снять
флажки с неподтверждённых изменений и загрузить выбранные файлы. Обработка
также умеет сохранять таблицу изменённых объектов, список изменённых файлов
и журнал автоформатирования.

## Программный вызов

Объект обработки экспортирует функцию:

```bsl
Статус = ОбработкаОбъект.АвтоформатироватьКод(ПараметрыАвтоформатирования);
```

Без структуры параметров используются текущая база и временный каталог;
выгрузка и обратная загрузка XML включены, а обновление БД и помещение
в хранилище отключены. Функция возвращает пустую строку при успехе или текст
ошибки.

Основные параметры: `ПутьКБазе`, `Пользователь`, `Пароль`,
`КаталогВыгрузкиXML`, `ВыгружатьКонфигурациюВФайлыXML`,
`ЗагружатьКонфигурациюИзФайловXML`, `ОбновитьКонфигурациюБД`,
`КаталогХранилища`, `ПользовательХранилища`, `ПарольХранилища`,
`ПомещатьИзмененныеОбъектыВХранилище`, `СтрокаЗапускаПлатформы`,
`СписокЗахваченныхОбъектов`, `ПутьКФайлуРезультатов` и `ЗначениеФлагов`.

## Пакетный запуск

Настройте обработку интерактивно и выполните команду
**Выгрузить параметры автоформатирования**. Она создаёт XML или ZIP с точной
структурой параметров текущей версии.

```bat
"C:\Program Files\1cv8\8.3.27.XXXX\bin\1cv8.exe" ENTERPRISE ^
  /F "C:\Bases\Development" ^
  /N "Администратор" ^
  /Execute "C:\Tools\АвтоформатированиеКодаИЛокализация.epf" ^
  /C "C:\Tools\Config.xml"
```

Для серверной базы используйте `/S "server\development"` вместо `/F`.
Обработка принимает XML и ZIP. Путь после `/C` читается до первого символа
`|`, если он присутствует.

Минимальный пример:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<НастройкиАвтоформатирования>
  <ПутьКБазе>File="C:\Bases\Development";</ПутьКБазе>
  <КаталогВыгрузкиXML>C:\Temp\autoformat</КаталогВыгрузкиXML>
  <ВыгружатьКонфигурациюВФайлыXML>true</ВыгружатьКонфигурациюВФайлыXML>
  <ЗагружатьКонфигурациюИзФайловXML>false</ЗагружатьКонфигурациюИзФайловXML>
  <ОбновитьКонфигурациюБД>false</ОбновитьКонфигурациюБД>
  <СтандартыДляАвтоформатирования>
    <Стандарт>КаноническийВидКлючевыхСловЗапроса</Стандарт>
    <Стандарт>КаноническийВидКлючевыхСловЯзыка</Стандарт>
  </СтандартыДляАвтоформатирования>
</НастройкиАвтоформатирования>
```

Надёжнее экспортировать настройки той же версией EPF, затем изменить пути
и набор правил. Экспорт содержит все реквизиты обработки. Пароли базы
и хранилища могут попасть в файл — не добавляйте его в Git.

## Ограничения

- Обработка требует платформу не ниже 8.3.6.
- Она меняет BSL, XML метаданных и форм.
- Некоторые правила в EPF связаны только со стандартом целиком.
- Дерево правил имеет значения по умолчанию — проверяйте выбор перед запуском.
