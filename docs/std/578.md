# Рабочий стол

Рабочий стол - личный помощник пользователя.
Рабочий день часто начинается именно с него.

Рабочий стол должен быстро вводить в курс дел,
отвечая на вопросы, например:

- что нужно сделать сегодня;
- что появилось нового;
- на что следует обратить внимание;
- каково состояние важных для пользователя сведений.

## Что размещать на рабочем столе

При определении состава форм,
доступных на рабочем столе,
следует исходить из набора ролей конфигурации.

Каждой роли может соответствовать одна или несколько форм рабочего стола.
Так как пользователь обычно относится сразу к нескольким ролям,
рабочий стол будет состоять из комбинации форм этих ролей.

При проектировании рабочего стола учитывайте:

- это первое,
  что пользователь видит при запуске программы,
  поэтому разрабатывать его нужно особенно тщательно;
- для командной панели рабочего стола допускается отключить флажок
  `Автозаполнение`
  и заполнять команды вручную;
- к формам рабочего стола стоит относиться
  как к микро-рабочим местам:
  они не должны быть перегружены,
  но должны давать богатые возможности;
- содержимое рабочего стола формируется автоматически в соответствии с ролями,
  а пользователь,
  как правило,
  исполняет несколько ролей.

!!! example "Примеры"

    Список задач может поддерживать `drag-and-drop`:
    при перетаскивании в него заказа покупателя
    система сразу предлагает создать новую задачу для коллеги
    со ссылкой на этот заказ.

    Менеджер по продажам может одновременно:
    заниматься продажами,
    вести личный учет времени,
    исполнять общефирменные поручения и приказы,
    анализировать рынок,
    вести исследования для развития направления.

## Что нужно учитывать при разработке

- Рабочий стол - обязательный раздел, его нельзя отключить.
- Проектируйте формы рабочего стола под существенно меньший размер,
  чем обычные формы.
- Не рекомендуется размещать на формах рабочего стола
  списки с горизонтальной полосой прокрутки.
- Не рекомендуется использовать в формах рабочего стола меню
  `Все действия`.
- По возможности не добавляйте команду вызова справки
  в формы рабочего стола.
- В командных панелях форм рабочего стола
  рекомендуется оставлять минимально необходимый набор команд.
- В командной панели списков
  не рекомендуется добавлять команды поиска и отмены поиска.
- Формы для рабочего стола проектируйте "с запасом":
  делайте их по умолчанию невидимыми,
  чтобы пользователь мог настраивать рабочий стол под себя.
- В командных панелях динамических списков
  рекомендуется размещать команду обновления.
- Состав форм рабочего стола проектируйте так,
  чтобы у реального пользователя
  (с учетом его ролей)
  по умолчанию было от `3` до `6` форм.
  Иначе рабочий стол будет выглядеть вырожденным или перегруженным.
- В списках на рабочем столе
  по умолчанию должно быть видно не менее `3,5` строк.

## Панель действий на рабочем столе

- Панель действий на рабочем столе рекомендуется делать
  только если можно достоверно выделить
  частотные и неконтекстные команды:
  - команды,
    которыми пользуются часто;
  - команды,
    для выполнения которых не нужно переключаться в другой раздел.
- Следите,
  чтобы панель действий не стала непонятной
  из-за разнородности команд,
  попавших в нее из разных разделов.
