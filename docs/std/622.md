###### #std622

# Флажки

Флажки используйте
для реквизитов типа `Булево`.

Подписи к флажкам
оформляйте по правилам ниже.

###### 1.

Подпись располагайте
справа от флажка.

###### 2.

Если для флажка нужна подсказка,
ее располагайте
либо снизу,
либо справа от подписи.

!!! success "Правильно"

    ![Неправильное расположение подсказки](622.2.3.wrong.png){ width="407" }

</div>

###### 3.

Первая буква подписи флажка
должна быть заглавной.

<div class="std-good-bad-pair" markdown="1">

!!! success "Правильно"

    ![Подпись с заглавной буквы](622.3.1.right.png){ width="122" }

!!! failure "Неправильно"

    ![Подпись не с заглавной буквы](622.3.2.wrong.png){ width="122" }

###### 4.

Подписи у флажков
делайте позитивными
(без отрицания).

<div class="std-good-bad-pair" markdown="1">

!!! success "Правильно"

    ![Позитивная подпись](622.4.1.right.png){ width="145" }

!!! failure "Неправильно"

    ![Подпись с отрицанием](622.4.2.wrong.png){ width="145" }

</div>

Если при этом требуется,
чтобы флажок по умолчанию
был установлен,
у реквизита объекта метаданных
следует установить свойство
`Значение заполнения`
в значение `#!bsl Истина`.

Подпись с отрицанием
следует применять
только в исключительных случаях,
когда это выглядит естественно
или обусловлено историческими причинами.

Например:
`Не показывать это окно`,
`Больше не спрашивать`,
`Недействительный`
(для учета неактуальных пользователей).

###### 5.

Текст подписи
должен быть понятным и кратким.

Не следует делать подпись,
состоящую из двух и более предложений.

###### 6.

Подпись к флажку
определяет только один вариант,
второй остается неявным.

Поэтому текст подписи
следует подбирать так,
чтобы у пользователей
не возникало сомнений,
каким будет второй вариант.

<div class="std-good-bad-pair" markdown="1">

!!! success "Правильно"

    ![Однозначная подпись флажка](622.6.1.1.right.png){ width="231" }

!!! failure "Неправильно"

    ![Неоднозначная подпись флажка](622.6.1.2.wrong.png){ width="215" }

</div>

Если такую подпись
подобрать не удается,
лучше использовать радиокнопки,
у которых явным образом
заданы названия вариантов.

<div class="std-good-bad-pair" markdown="1">

!!! success "Правильно"

    ![Радиокнопки вместо неоднозначного флажка](622.6.2.1.right.png){ width="110" }

!!! failure "Неправильно"

    ![Флажок вместо явного выбора вариантов](622.6.2.2.wrong.png){ width="110" }

</div>

###### 7.

Если на форме
присутствует группа флажков,
в подписях которых
используется общий текст,
его следует выносить
в заголовок группы.

<div class="std-good-bad-pair" markdown="1">

    ![Общий текст дублируется в подписях флажков](622.7.2.wrong.png){ width="305" }

</div>
