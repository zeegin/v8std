###### #std500

# Общие правила построения интерфейсов

###### 1.

Для каждой укрупненной группы пользователей
(рабочего места)
в конфигурации
рекомендуется определять
отдельный интерфейс:

- главное меню;
- набор панелей инструментов;
- состав панелей инструментов.

###### 2.1.

Главное меню должно обеспечивать доступ
ко всем формам,
которые требуются пользователю
для работы и сервисных функций.

В подменю
с большим числом элементов
группы элементов
следует разделять разделителями.

Максимальное число элементов
в одной группе
не более `7`.

###### 2.2.

При любом положении выбора действия
из главного меню
его размер должен быть таким,
чтобы полностью умещаться на экране
при минимальном разрешении,
на которое рассчитана конфигурация.

###### 2.3.

По умолчанию пункты меню
для доступа к:

- справочникам;
- документам;
- планам видов характеристик;
- планам счетов;
- планам видов расчетов;
- регистрам

должны вызывать
формы списков этих объектов.

###### 3.1.

Интерфейс следует проектировать так,
чтобы:

- группе пользователей
  был доступен необходимый набор действий;
- не предоставлялся доступ
  к действиям,
  на которые у пользователя нет прав.

Наиболее часто выполняемые действия
лучше располагать
в наиболее доступных местах,
редко используемые - наоборот.

###### 3.2.

Однотипные блоки меню
и панелей инструментов
в разных интерфейсах
желательно делать похожим образом.

###### 4.

В каждой конфигурации
обязательно должны быть интерфейсы:

- `Общий`;
- `Полный`.
