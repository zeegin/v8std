###### #std616

# Формы списков

- Состав и порядок колонок
  должны определяться прогнозируемым сценарием заполнения.
- Колонки следует отбирать и группировать в несколько этажей так,
  чтобы обходиться без горизонтальной полосы прокрутки.
- Фиксируйте колонки там,
  где это уместно.
- По умолчанию
  рекомендуется делать активной колонкой ту,
  по которой наиболее вероятен поиск.
- Не рекомендуется использовать режим выделения строки `Строка`.

!!! note "Примечания"

    В многоэтажных списках
    сложнее воспринимать данные при беглом чтении.

    Пример колонок,
    которые часто имеет смысл фиксировать:
    `№` и `Номенклатура` в списках документов.

    В динамических списках
    допустимо добавлять колонки "с запасом"
    и делать их по умолчанию невидимыми,
    чтобы пользователь позже настроил список под себя.
    Для табличных частей
    эту рекомендацию применять не следует.

    Для табличной части списка товаров
    активной колонкой по умолчанию
    обычно является `Товар`.
