###### #std787

# Вычисление количества записей в запросах

###### 1.

При вычислении количества записей в запросе всегда используйте функцию `#!sdbl КОЛИЧЕСТВО`, а не `#!sdbl СУММА`.

Иначе при количестве записей 10 млн и более возможно переполнение, связанное с разрядностью чисел по умолчанию, используемых СУБД платформой 1С:Предприятие.

<div class="std-good-bad-pair" markdown="1">

!!! success "Правильно"

    ```sdbl
    КОЛИЧЕСТВО(*)
    КОЛИЧЕСТВО(<Поле>)
    ```

!!! failure "Неправильно"

    ```sdbl
    СУММА(1)
    ЕСТЬNULL(СУММА(1), 0)
    ```

</div>

###### 2.

Если количество нужно вычислять условно и `#!sdbl КОЛИЧЕСТВО` применить нельзя, расширяйте разрядность числа по умолчанию через `#!sdbl ВЫРАЗИТЬ` (например, до 17 разрядов).

<div class="std-good-bad-pair" markdown="1">

!!! success "Правильно"

    ```sdbl
    ВЫБРАТЬ
        КОЛИЧЕСТВО(*) КАК Количество,
        СУММА(ВЫБОР
            КОГДА Номенклатура.ЭтоГруппа
                ТОГДА ВЫРАЗИТЬ(1 КАК ЧИСЛО(17, 0))
            ИНАЧЕ 0
        КОНЕЦ) КАК КоличествоГрупп,
        СУММА(ВЫБОР
            КОГДА НЕ Номенклатура.ЭтоГруппа
                ТОГДА ВЫРАЗИТЬ(1 КАК ЧИСЛО(17, 0))
            ИНАЧЕ 0
        КОНЕЦ) КАК КоличествоЭлементов
    ИЗ
        Справочник.Номенклатура КАК Номенклатура
    ```

!!! failure "Неправильно"

    ```sdbl
    ВЫБРАТЬ
        КОЛИЧЕСТВО(*) КАК Количество,
        СУММА(ВЫБОР
            КОГДА Номенклатура.ЭтоГруппа
                ТОГДА 1
            ИНАЧЕ 0
        КОНЕЦ) КАК КоличествоГрупп,
        СУММА(ВЫБОР
            КОГДА НЕ Номенклатура.ЭтоГруппа
                ТОГДА 1
            ИНАЧЕ 0
        КОНЕЦ) КАК КоличествоЭлементов
    ИЗ
        Справочник.Номенклатура КАК Номенклатура
    ```

</div>
