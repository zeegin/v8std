###### #std632

# Использование констант

Запись константы следует выполнять вне транзакций, так как запись может стать узким местом при конкурентной работе.

!!! failure "Неправильно"

    ```bsl
    // Увеличиваем счетчик проведенных документов в константе
    Процедура ОбработкаПроведения(Отказ, РежимПроведения)
        ТекущееЗначение = Константы.СчетчикПроведенныхДокументов.Получить();
        Константы.СчетчикПроведенныхДокументов.Установить(ТекущееЗначение + 1);
    КонецПроцедуры
    ```

Во время записи константы работа других сеансов приостанавливается, если они одновременно пытаются записать эту же константу. Подробнее о причинах избыточных блокировок и методах оптимизации см. базу знаний [«Технологические вопросы крупных внедрений»](http://kb.1c.ru).

При этом недопустимо решать проблему блокировок тотальным кешированием констант:

- в [#std413: параметрах сеанса](413.md);
- в [#std724: общих модулях с повторным использованием возвращаемых значений](724.md).
