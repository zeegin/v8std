###### #std574

# Группировка команд

!!! warning "Устаревший стандарт"
    Этот стандарт устарел. Используйте [#std714: Навигация внутри раздела (8.3)](714.md).


- Если команд в группе `Обычное` много,
  группируйте их по смыслу.
  Для этого пересматривайте содержимое подсистемы
  и разбивайте его так,
  чтобы в группе было от `4` до `9` команд.
- Состав команд проектируйте так,
  чтобы по умолчанию в панели навигации
  не было полосы прокрутки.
- По возможности избегайте появления в панели навигации
  групп,
  состоящих из одной команды.
- Не рекомендуется делать глубину вложенности групп
  более `2`.
- Учитывайте,
  что по умолчанию все группы раскрыты.
- Названия групп в панели навигации
  не должны пересекаться с системными группами,
  например:
  `Создать`, `Отчеты`, `Сервис`, `См. также`.

!!! note "Примечания"

    Критерием количества команд
    может служить оценка емкости кратковременной памяти человека
    `7 ± 2`.

    Нечастотные команды можно скрывать:
    при необходимости пользователь включит их сам.

    При построении командного интерфейса учитываются роли,
    поэтому для отдельных пользователей
    группа может стать вырожденной
    (состоять из одной команды).

    На втором уровне вложенности
    ширина текста команды по умолчанию меньше
    (примерно `36` символов).
