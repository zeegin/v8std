###### #std567

# Панель разделов

!!! warning "Устаревший стандарт"
    Этот стандарт устарел. Используйте [#std712: Панель разделов (8.3)](712.md).


**Панель разделов** - оглавление конфигурации.

Разделы должны соответствовать реальным областям деятельности
или участкам работ так, как их понимают пользователи.

Состав разделов рекомендуется проектировать так,
чтобы в типовых комбинациях ролей
не появлялась полоса прокрутки.

Для этого емкость панели должна быть
не более `8-10` разделов.

## Что учитывать при проектировании

- Панель разделов - "лицо" программы.
  На этапе освоения она особенно важна,
  поэтому состав и порядок разделов
  нужно проектировать тщательно.
- При выполнении задач,
  связанных с определенной деятельностью,
  пользователь должен проводить большую часть времени
  в одном разделе.
  Раздел - устойчивый режим работы.
  Задачи одной деятельности
  должны решаться в рамках одного раздела.
  Переключения между разделами нужно минимизировать.
- Интерфейс раздела проектируйте так,
  чтобы он содержал все необходимые команды и функции.
  Частотные и важные команды
  следует поднимать наверх:
  в начало панели навигации и панели действий.
- При проектировании панели
  учитывайте доступность разделов (подсистем) по ролям.
- Для интерфейса администратора
  допустимо наличие полосы прокрутки в панели.
- По умолчанию разделы располагаются по алфавиту.
  Рекомендуется задавать порядок по приоритету:
  от наиболее частотных и значимых
  к второстепенным и редким.
- Последним всегда должен быть раздел
  для администрирования,
  настройки и сервисных действий.
- Не рекомендуется делать раздел с названием `Сервис`.

!!! example "Пример"

    Название `Сервис` пересекается
    с меню `Сервис`
    и группой `Сервис` в панели действий.

    Возможные альтернативы:
    `Сервисные возможности`,
    `Дополнительные возможности`,
    `Прочее`,
    `Сервисные функции`,
    `Служебные функции`.
