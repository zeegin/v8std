###### #std711

# Общие принципы построения командного интерфейса

Командный интерфейс это средство навигации пользователя по функциональности конфигурации.

Он включает:

- панель разделов;
- панель функций текущего раздела;
- меню функций;
- команды навигации и действий.

Каждый элемент решает свою задачу, вместе они формируют пространство команд для пользователя.

Состав и расположение панелей можно настраивать под специфику прикладного решения.
При проектировании соблюдайте следующие принципы.

###### 1.

Количество элементов в панелях и меню должно быть таким, чтобы при стандартном разрешении экрана они помещались без прокрутки.

###### 2.

Элементы внутри панелей и меню располагайте по убыванию важности и частоты использования.
Самые приоритетные пункты и команды размещайте первыми.

###### 3.

Не располагайте рядом команды и пункты меню, в названиях которых совпадают первые символы.

<div class="std-good-bad-pair" markdown="1">

!!! success "Правильно"

    ```text
    Регламентированные отчеты
    Отчеты для руководителя
    ```

!!! failure "Неправильно"

    ```text
    Регламентированные отчеты
    Регламентные операции
    ```

</div>

###### 4.

Проектируйте командный интерфейс так, чтобы он повышал эффективность повседневной работы и ускорял освоение программы.
Для этого учитывайте мнение пользователей и то, как они ожидают видеть группировку команд.
