# Правила работы с репозиторием

- Исправляй причину, а не последствие.
- Находи противоречия в рассуждениях пользователя и результатах работы, не замалчивай их, а явно подсвечивай.
- Ищи логические ошибки и указывай на них.
- После завершения работы сообщай, сколько токенов было потрачено.
- По умолчанию работай в основном checkout репозитория. Не создавай и не используй Git worktree без явного указания пользователя.
- Прямые коммиты и push в ветку `main` разрешены. Отдельная ветка или pull request нужны только по явному указанию пользователя.
